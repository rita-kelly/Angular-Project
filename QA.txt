Pokedex Trainer Dashboard - Client Requirement Q&A
==================================================

Purpose
-------
This file translates the client description into evaluator-style questions and
answers. It is meant to help a reviewer quickly understand:

1. What the client wanted.
2. Where the app implements each requirement.
3. Why certain technical choices were made.
4. Which items are complete, partial, or intentionally handled differently.


Client Intent Summary
---------------------
Q: Did the client ask for only a Pokemon list?
A: No. The client asked for a single-page Angular 19 trainer dashboard where a
   Pokemon trainer can browse Pokemon, build teams, track battles, and manage
   a trainer profile using GraphQL.

Q: What is the real target of the assessment?
A: The target is to demonstrate Angular 19, GraphQL, RxJS state management,
   Signals, charts, advanced forms, table UX, video/audio embedding, testing,
   code comments, and at least two advanced bonus tasks.


1. GraphQL Integration
----------------------
Q: Which public GraphQL API is used?
A: The public Pokemon API endpoint is:
   https://beta.pokeapi.co/graphql/v1beta

   Main files:
   - src/app/graphql/graphql-endpoints.ts
   - src/app/graphql/apollo.providers.ts
   - src/app/api/pokemon-api.service.ts

Q: Which local GraphQL API is used?
A: The local mock API is json-graphql-server using db.js.

   Current repo default:
   - http://localhost:4100/graphql

   Script:
   - npm run local:graphql

   Note: The client brief says port 4000. This repo uses 4100 by default
   because port 4000 was already occupied during development. An alternate
   script exists:
   - npm run local:graphql:4000

Q: Where is the paginated Pokemon list query implemented?
A: src/app/api/pokemon-api.service.ts
   - getPokemonPage(limit, offset)
   - Uses GraphQL variables.
   - Uses retry with delay.

Q: Does the paginated list fetch sprites?
A: Partially. The table model supports spriteUrl and the template renders an
   image if spriteUrl exists. For performance, list rows currently avoid loading
   remote sprite images on first paint and show a stable #id fallback. Full
   artwork is loaded in the Pokemon detail query.

   Relevant files:
   - src/app/api/pokemon-api.service.ts
   - src/app/features/pokedex/pokedex-page.component.html
   - src/app/features/pokedex/pokemon-detail-panel/pokemon-detail-panel.component.html

Q: Where is the single Pokemon detail query implemented?
A: src/app/api/pokemon-api.service.ts
   - getPokemonDetails(id)
   - Fetches stats, types, abilities, moves, sprites/artwork, and evolution
     chain data.

Q: Where are Pokemon type matchups fetched?
A: src/app/api/pokemon-api.service.ts
   - getTypeEfficacies()

   src/app/state/pokemon.store.ts
   - loadTypeEfficacies()

   src/app/state/pokemon.selectors.ts
   - typeEffectivenessMap$()

Q: Is type effectiveness used in the UI?
A: Yes, via a bonus directive:
   - src/app/shared/type-highlight.directive.ts
   - src/app/features/pokedex/pokedex-page.component.html

   Note: The directive uses a local type-effectiveness map for immediate row
   highlighting. The GraphQL type-efficacy fetch and shareReplay selector are
   also present for API-backed matchup data.

Q: Where are local trainer/team/battle/battle-log queries implemented?
A: src/app/api/trainer-api.service.ts
   - getTrainers()
   - getTrainer(id)
   - getTeams(trainerId)
   - getBattles(trainerId)
   - getBattleLogs()

Q: Where are local mutations implemented?
A: src/app/api/trainer-api.service.ts
   - createTeam(data)
   - updateTeam(id, data)
   - deleteTeam(id)
   - updateTrainer(id, data)
   - logBattle(data)

Q: Are create/edit/delete team mutations all supported?
A: The API service and TrainerStore support create, update, and delete.

   UI status:
   - Create team is exposed in Team Builder.
   - Update/delete team are implemented at API/store level but are not exposed
     as visible edit/delete buttons in the current UI.

   If the evaluator requires visible edit/delete controls, this is a follow-up
   UI task.

Q: Where is the battle result mutation exposed?
A: Route:
   - /battles

   Files:
   - src/app/features/battles/battles-page.component.ts
   - src/app/state/trainer.store.ts
   - src/app/api/trainer-api.service.ts

Q: Where is the trainer profile mutation exposed?
A: Route:
   - /profile

   Files:
   - src/app/features/profile/profile-page.component.ts
   - src/app/state/trainer.store.ts
   - src/app/api/trainer-api.service.ts

Q: Why is polling used instead of true WebSocket GraphQL subscriptions?
A: The local mock server is json-graphql-server. It exposes HTTP GraphQL
   queries and mutations but does not provide a WebSocket subscription
   transport. Because of that, the app simulates a live subscription by polling
   battle logs every 5000ms with RxJS interval(5000) + switchMap(...), then
   emitting only new entries into the live feed.

   This is clearly commented in:
   - src/app/state/trainer.store.ts
   - connectLiveBattleLogFeed()

Q: Where is interval(5000) + switchMap implemented?
A: src/app/state/trainer.store.ts
   - connectLiveBattleLogFeed(pollMs = 5000)

Q: Why is the db.js collection named battleLogs instead of battle_log?
A: json-graphql-server had schema/resolver issues with the snake_case
   collection name. The repo uses battleLogs to keep the generated GraphQL
   schema valid. The comment is in db.js.


2. RxJS State Management
------------------------
Q: Did the app use NgRx, Akita, or NgXS?
A: No. It uses custom stores built with BehaviorSubject.

Q: Where are the custom stores?
A:
   - src/app/state/pokemon.store.ts
   - src/app/state/trainer.store.ts

Q: Where are selectors?
A:
   - src/app/state/pokemon.selectors.ts
   - src/app/state/trainer.selectors.ts

Q: Is BehaviorSubject the core state holder?
A: Yes.

   Files:
   - PokemonStore: private stateSubject = new BehaviorSubject(...)
   - TrainerStore: private stateSubject = new BehaviorSubject(...)

Q: Are selectors derived Observable streams?
A: Yes.

   Examples:
   - pokemonSearchResults$()
   - pokemonFilterByType$()
   - typeEffectivenessMap$()
   - winRate$()
   - battleResultsPerMonth$()
   - teamStats$()

Q: Is Pokemon search debounced?
A: Yes for the Pokedex selector:
   - src/app/state/pokemon.selectors.ts
   - debounceTime(300) -> distinctUntilChanged() -> switchMap(...)

   Note: Team Builder search currently filters cached Pokemon through Signals
   and does not re-query the public API per keystroke. If strict typeahead API
   querying is required, that is a possible follow-up.

Q: Is shareReplay(1) used?
A: Yes.

   Files:
   - src/app/state/pokemon.selectors.ts
   - src/app/state/trainer.selectors.ts

Q: Is retry(3) with delay used for PokeAPI calls?
A: Yes.

   File:
   - src/app/api/pokemon-api.service.ts

Q: Are subscriptions cleaned up?
A: Yes. Components use DestroyRef and takeUntilDestroyed().

   Examples:
   - src/app/features/pokedex/pokedex-page.component.ts
   - src/app/features/team-builder/team-builder-page.component.ts
   - src/app/features/battles/battles-page.component.ts
   - src/app/features/profile/profile-page.component.ts
   - src/app/features/pokedex/pokemon-detail-panel/pokemon-detail-panel.component.ts

Q: Is optimistic update with rollback implemented?
A: Yes for team creation.

   File:
   - src/app/state/trainer.store.ts
   - createTeamOptimistic()

12. Known Partial Items / Follow-Up Questions
---------------------------------------------
These are the items to know before a strict final client review.

Q: Is team edit/delete visible in the UI?
A: Not currently. API and store methods exist, but visible edit/delete controls
   are not exposed in the Team Builder/Profile UI.

Q: Is a pie/doughnut team type distribution chart visible?
A: Not currently. This is the biggest remaining chart-specific gap.

Q: Are list sprites loaded in the Pokedex table?
A: The table supports sprite images, but the current list data uses a fallback
   id badge for performance. Detail artwork is loaded in the side panel.

Q: Is the Team Builder autocomplete re-querying the public API per keystroke?
A: Not currently. It filters cached Pokemon data. Pokedex search uses the
   required debounceTime/distinctUntilChanged/switchMap selector.

Q: Is the total base-stat filter a slider?
A: No. It is implemented as min/max numeric inputs.

Q: Is true GraphQL WebSocket subscription implemented?
A: No. The local server does not support WebSocket subscriptions. Polling is
   implemented and clearly commented, as requested.
